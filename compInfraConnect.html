Troubleshooting network connections

Problems with no connectivity or intermittent connectivity

Guidelines to follow when troubleshooting network problems

  • To check connectivity,ping the local router,then try pinging other devices on the network.No connectivity might be caused by network cabling or it's connection or wireless switch not turned on or a bad network adapter or network settings
  • Determine whether other computers on the network are experiencing the same issues you have on one computer.It the entire network is down,the problem is not isolated to the computer you are working on
  • To find out if a computer with limited or no connectivity was able to initially connect to a DHCP server onthe network,check for an (APIPA) on an IPv4 (169.254) network and check for link local on IPv6 (FE80).A computer assigns APIPA address if it is unable to connect to DHCP server.Using ipconfig command to find out the IP address
  • Check the status indicator lights on the network ports for connectivity and activity
  • Use loopback plug to verify that each port is working.The loopback plug can test the ports on a computer,wall jack,patch panel,switch,router,or other network device that is turned on,if you find a bad port,try a different port on a swtch or router or patch panel.Try resetting a switch or router.For a router try updating it's firmware.you might need to replace the device
  • For short,straight-through cables that connect a computer to a wall jack or other nearby device,exchanging the straight-through cable for a known good one might be easier and quicker than using a cable tester to test the cable.For longer cables,especially inside the walls,ceiling and raised floors,consider that cable length might exceed the recommented maximum length
  • Use cable tester to verify that a cable permanently installed inside the wall is good.To test the cable,you first have to disconnect it from the computer,patch panel,switch,router at both ends of the cable.Common problems with networks are poorly wired termination in patch panels and wall jacks.If the cable proves bad,first try reinstalling the two jacks before you replace the cable
  • Test for access to other network resources,such as file shares,network printers,if you can successfully ping the devices hosting those resources,but you can't access the resources themselves,this indicate the problem at a higher TCP/IP layer such as permission configurations or problems at the host for the network service
  • If you can access some but not all devices on the network,limited connectivity might be caused by cables or switch on the network or a problem at the other computers you are trying to reah
  • To test for internet access, use browser to surf the web .Problems with no internet access can be caused by cables,a SOHO router,a broadband modem or problems at the ISP

    • When a computer has no internet connectivity or intermittent connectivity,begin by checking hardware and then move on to checking windows network settings

  • Check the status indicator on the NIC or motherboard Ethernet port.Steady light indicates connectivity and blinking light indicates activity.Check light on the router or switch from the other end.Try different port on the device.If router or switch is in server closet and ports are not labelled,you can try using loopback plug to find which port the computer is using.If you don't see either light,this problem must be resolved before you consider OS or application problems
  • Check network cable connections at both ends.Check if the port on the motherboard that is connected is disabled,it might need to be connected to the network port provided by a network card .A cable tester can verify that the cable is good or if it is the correct cable.Try different cable
  • For wireless networking,make sure the wireless radio on the computer is turned on.If you have no connectivity,limited connectivity or intermittent connectivity,consider moving the computer to a new position in the hotspot or shift the wireless access point's location.Use a wireless locator to find the best position.Try turning the wireless connection off,wait a few seconds then turn it back on.Rebooting a computer might help to solve the problem.Problem with a low RF( radio frequency) signal can sometimes be solved by moving the laptop or connecting to a different wireless access point that has a stronger RF signal
  • If the problem ios still there after you have checked all connections and wireless radio,turn to windows to repair the network connection.Use the following methods:
      ∠ Use ipconfig/release followed by ipconfig/renew
      ∠ In the network and sharing center,click Troubleshoot problems to access a diagnostic tool for windows network connectivity
      ∠ Return to the network and sharing center, click Change adapter settings,right-click the connection and click Disable .Then right-click again and click Enable

    • If the problem still persist,,dig deeper,maybe the problem is network adapter drivers.To solve problems with device drivers,which might also be related to a problem with the NIC,follow these steps:

  • Make sure network adapter and it's drivers are installed,by checking for the adapter in the Device Manager,it should report the device is working
  • If error reports,try updating the device drivers.If the drivers still install with errors,look for manufacturer's website or the installation CD that came bundled with the adapter for diagnostic software that might help diagnose the problem
  • Try uninstalling and reinstalling the network adapter
  • If Device manager still reports errors,tyr runnign anti-malware software and updating windows.Then try replacing your network adapter.If that doesn't work,the problem might be a corrupted windows installation
    • Problems with internet connectivity

    If you have local connectivity,but not internet access,do the following

  • Try recycling the connection to the ISP.Try these steps:
      ∠ Go to power source of the cable modem,DSL model or other device that you use tho connect to your ISP,and unplug the power,unplug the router.Wait about five minutes for the connection to break at the isp
      ∠ Plug in the cable modem,DSL modem,or other ISP device.wait until the lights settle,then plug in your router
      ∠ ON any computer on ypour network,use the network and sharing center to repair the network connection.Open your browser and try to browser and try to browse to some websites
  • Check to make sure your television works, that's if you are using cable modem.The service might be down.For DSL connection,check to make sure your landline phone gives a dial tone,the phone might be down
  • To eliminate router as the source of the problem,connect one computer directly to the broadband modem.If you access the internet,you have have proven the problem is with the router or cable to it.Connect the router back into the network,and check all the router settings.The problem might be with DHCP,the firewall settings or port forwading.Try updating the firmware on the router.If you are convinced all settings on the router are correct,but the connection to your ISP works without the router and does not work with the router,it's time to replace the router
  • To eliminate DNS as the problem,follow these steps:
      ∠ Try substituting a domain name for the IP address in a ping command.Ping google's DNS server
      ping 8.8.8.8 then ping domain name: ping google.com
      If both pings work,then you can conclude that DNS works.If an IP address works but domain name does not work,the problem lies with DNS
      ∠ Try pinging your DNS server.To find out the IP address of your DNS server,use the nslookup command without parameters or open the firmware utility of your router and look on a status screen
      ∠ Try changing your DNS server to public DNS server.In the network connections window, right-click the active connection and click Properties.Click Internet Protocol Version 4(TCP/IPv4) and click Properties.Select Use the following DNS server address
  • If you are having a problem accessing a particular computer on the internet,try using the tracert command tracert mwserve.co.za
    The results show computers along the route that might be giving delays
  • If one computer does not have internet access,make suire the MAC address filtering on the router is disabled or computer is allowed access
  • If the problem is with thw router firewall or windows firewall.Try checking the firewall and change settings
  • If you cannot access the internet,contact your ISP
    • Using TCP/IP utilities to solve connectivity problems

    Connectivity problems can be caused by TCP/IP cofiguration and can be solved using windows TCP/IP utilities.Follow these steps to verify that local computer is communicating over the network:
    1. Using the network and sharing center or ipconfig connand, try to release the current IP address and lease a new address.Thsi process solves the problem of an IP conflict with other computers on the network when using DHCP and dynamic IP address or your computer's failure to connect to the network
    2. For static IP address,consider that duplicate static iP addresses may have been assigned to hosts on the network.You need to go to each computer or device that is using static IP addresses and verify that no two addresses are the same
    3. To find out if you have local connectivity, try to ping another computer on the network.Find out if you have internet connectivity and DNS is working,try to ping a computer on the internet using it's host address.Enter ping mwserve.co.za.If this last command doesn't work,try tracert command to find out if the problem is outside or inside your network.Enter tracert mwserve.co.za
    4. Enter ipconfig /all to verify the IP address,subnet mask and default gateway.For dynamic IP addressing,if the computer cannot reach the DHCP server,it assigns itself an APIPA address which begins with 169.254, suspect that the computer is not able to reach the network or DHCP server is down
    5. Try loopback address test.Use the ping 127.0.0.1 command.Your computer should respond,if you get an error,assume the problem is TCP/IP settings on your computer.Compare the configuration with that of a working computer on the same network
    6. If you have problems with slow transfer speed,a process is hogging network resources.Use the netstat -b command to find out if the program you want to use to access the network is actually running.
    7. Firewall settings might be wrong,Are port forwading settings on the router and in windows firewall set correctly?
    8. If you are having problems getting a network drve map to work,try makign the connection with the net use command Net use C; \\ cmputername\folder
    To disconnect a mapped network drive,use this command: net use c: /delete

    Slow transfer speeds

    You have internet connectivity but the file transfer speed is slow,lagging websites or network-based applications and possibly a variety of error messages.It's best to look first for simple problems causing these delays.Using your critical thinking skills,start at the hardware level of the TCP/IP model and work your way up the layers:

  • There is unofficial layer affectionately called User layer. If the user is present,interview the user and have the user reproduce the problem while you watch.Ask questings such as "When did the problem start? and what changed since the problem started.Determine if you might have some tips for how to access network resources more effeciently and effectively.
  • Link layer Check cables for secure connections,status lights for consistent connectivity and network devices for indications of errors being reported.You can use device manager to check the NIC's speed and cuplex settings on Ethernet connections,ensuring that Auto Negotiate is selected
  • Internet layer Ping the computer's loopback address,default gateway and other devices on the network and server on the internet.Watch for indications of delayed respponses and use tracert if needed to help identify the exact location of the slow down
  • transport layer Check firewall settings for blocked ports,also confirm QoS settings are configured as expected and consider experimenting with different sittings to see if that improves network performance
  • Alpplication layer If everything below application is working,you need to focus on troubleshooting efforts on application installation,configuration and compatibility concerns.Try same operations using defferent but similar applications.Try using a different browser or different FTP cllient application.Keep in mind that both server and the client might need some troubleshooting to identify and solve an application-layer problem
    • Wi-Fi network Not found
    Wireless access point broadcasts an SSID.Your computer should easily find the SSID oa a Wi-Fi network and connect after entering the ssucurity key,if one is needed.If oyur computer does not detect an SSID that you know should be available,it might be because the SSID is hidden.If you know the SSID and security key,you can still connect to the network by following these steps:
    1. Open the Network and Sharing Center
    2. Click Set up a new connection or network
    3. Select Manually connect to a wireless network and click Next

    4. Enter the network name,choose the security type and enter the security key .Click Next
    5. Your wireless network is set up and you should be connected. Click Close
