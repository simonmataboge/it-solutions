Understanding TCP/IP and windows networking

When two computers communicate in a local network or internet,the communication happens at four level: the hardware,Operating system,application on each network and network.Communication begins when the computer tries to find the other computer on the local on remote network.When using web browser on a client to request a webpage from the web server,The client computer looks for the web server,the protocols for communication are established,and then the request is made and answered.

Layers of network communication

When two computers or devices communicate,they must be using the same protocols so that the communication makes sense.The group or suite of protocols used is called TCP/IP

TCP/IP model for network communication

Network communication starts when a browser(An application) requests a web page from the web server(another application).The layers of communication are shown with red arrows.The request is passed on to the OS,which passes the request to the network card,which passes the request on to the network.When the request reaches the network card on the server,the network card passes the request on to the OS and then the OS passes it on to the application(web server)

Layer Addressing Description
Layer 4:Application Layer Application-to-application communication is managed by the OS,using protocols specific to the application(HTTTP,Telnet,FTP,TCP).This layer of communication happens after the OS have made a connection at the transport layer
Layer 3:Transport Layer Port numbers Host-to-host communication,managed by the OS,primarily using TCP and UDP protocols
Layer 2:Internet Layer IP addresses Host-to-host on the local network or network-to-network communication,managed by OS and network devices
Layer 1:Link Layer MAC addresses Device-to-device on local network,managed by firmware on NICs.Layer 1 is also called the network interface layer or network access layer
  • Source Step 1: Application Layer
      When a client browser makes a request to a web server, the browser passes the request to the OS.The OS formats the message using the appropriate application protcol.Message is created and passed down in the TCP/IP stack of protocols to the Transport layer
  • Source Step 2: Transport Layer
      Transport layer adds information to the message to address the correct server application.Depending on the type of application,the protocol TCP or UDP adds the port assigned to the server application for HTTP communication.The message with transport data added is then passed down to the internet layer
  • Source Step 3: Internet Layer
      Internet layer is responsible for getting the message to the destination computer or host on the local network,an intranet or internet.An inntranet is any private network that uses TCP/IP protocols.A large enterprise might support an intranet that is made ip of several local networks.The primary protocol used at the internet layer is IP(internet protocol),which uses a 32-bit or 128-bit IP address to identify each host.(other internet protocols include, EIGRP,OSPF,BGP and ICMP).IP adds address information to the message and then passes it down to the link layer
  • Source Step 4: Link Layer
      Link layer is the physical network,hardware and it's firmware for every device connected to the network.NIC is part of this physical layer and is able to communicate with other NICs on the local network using NICs MAC address.The MAC address is a 48-bit unique number hard-coded on the card by it's manufacturer.For every device that connects to the network by the waay of it's NIC and MAC address.The NICs receive message from the IP,adds information for ethernet or Wi-Fi transmission and places the message on the network
  • Source Step 5: Network transmission
      IP at the internet layer is responsible to make sure the message gets from one network to the next until it reaches it's destination network.MAC address at the hardware link layer is only used to find a computer or other host on the local network,an IP address can be used to find a computer on a local network,anywhere on the internet or intranet
  • Source Step 6: Destination
      On the destination computer,it's NIC receives the messaage,strips off ethernet or Wi-Fi information at the link layer and passes the massage up to internet link.Internet layer strips off IP address information and passes the message to the correct port and on to the application layer.The application layer passes the message to the web server application

      1. IP address has 32-bits aor 128-bits.
      2. The Internet assigned Numbers Authority (IANA at iana.org ).is responsible for keeping track of assigned IP addresses
      3. IP addres is organised into four groups of 8-bits each,which are presented as four decimal munbers separated by periods,Such as 192.168.8.102
      4. The largest possible 8-bits is 11111111 which is equal to 255 in decimal
      5. Largest IP address in decimal is 255.255.255.255 which is
        11111111.11111111.11111111.11111111.11111111
      6. Each of the four numbers separated by periods is called octet
    IP Address How is it Used
    127.0.0.1 Indicates youor own computer and is called loopback address

    First part of an IP address idenfity the network and the last part identifies the host.
    The network portion of the IP address is used to locate the right local network.
    The host portion of the IP address is used to identify the one computer on the nwtwork that will receive the message.
    How does a computer or other network device know what part of an IP address identifies the network

    Subnet masks
      1. All the IP addresses assigned to a local network os subnet have the matching bits in the first part of IP address,these bits are called Network ID.
      2. The range of IP addresses assigned to a network might be 192.168.8.1-200.The first three octet 192.168.8 identify the network and the last octet 1 through 200 identify each host
      3. The last bits must be unique for each IP address on the network
      4. A subnet mask has 32-bits and is a string of 1s and followed by 0s. 11111111.11111111.11110000.00000000 which is 255.255.0.0 in decimal.
      5. IP address and subnet mask can be written in using a shorthand notation like 25.50.232.60/20. The /20 means that the first 20 bits in the IP address identify the network.this notation is called slash notation or CIDR (Classless Interdomain Routing) notation.     • How IPv6 addresses are used

    To improve routing capabilities and speed of communication,IPv6 changed the way IP addresses are used to find computers on the network.

      ∠ IPv6 address has 128-bits that are written as eight blocks of hexadecimal numbers separated by colons, like 2001:0000:0b80:0000:0000:00d3:9c5a:00cc
      ∠ Each block is 16 bits,the first block is written above is 2001 in hex,which can be written as 0010 0000 0000 0001 in binary
      ∠ leading zeroes in a four-charactor hex block can be eliminated. 2001:0000:b80:0000:0000:d3:9c5a:cc, leading zeroes have been removed from three of the hex blocks
      ∠ If blocks contain all zeroes,they can be written as double colons (::).The IP address above can be written in two ways
        ∠ 2001::b80;0000;0000d3:9c5a:cc
        ∠ 2001:0000:b80::d3:9c5a:cc

    To avoid confusion,only one set of double colons is used in an IPv6 address.The way computer communicate using IPv6 has changed the terminology used to describe TCP/IP communication.Here are few terms used in the IPv6 standards
      ∠ A link is a local area network (LAN) or wide area network (WAN).
      ∠ A node is any device that connects to the network,such as when virsual machine connects to the network or physical attachment such as when a network adapter connects to the wired network
      ∠ The last 64 bits or four blocks of anIPv6 address identify the interface and are called the interface ID or interface identifier.These 64 bits uniquely identify an interface on the local network
      ∠ Neighbor are nodes on the same network.IPv6 doesn't use broadcasting,it use multicasting,anycast and unicasting
      ∠ Multicast address is used to deliver messages to all nodes in a targeted,multicast group,such as video streaming from a server to multiple nodes on a network
      ∠ Anycast address is used by routers and can identify multiple destinations,a message is delivered only to the closest destination
      ∠ Unicast address is used to send message to a single node on a network.Three types of unicast addresses are link local addresses,unique local addresses and global addresses.A single interface might have more than one unicast address assigned to it at any given time:
        ∠ Link local address also called a link local unicast address or local address,can be used for limited communication with neighboring nodes in the same link.These local addresses are the same as APIPA in IPv4 addressing.They are addresses assigned toa computer by itself as oppsed to coming from a DHCPv6 server and are not guaranteed to be unique on the network.Most linklocal addresses begin with FE80::/64.This prefix notation means the address begins with FE80 followed by enough zeroes to make 64 bits.Link local addresses are not allowed on the internet or allowed to travel outside private networks.
        ∠ Unique local address is a private address assigned by DHCPv6 server that can communicate across subnets within the private network.They are used by network administrators when subnetting a large network.A unique local address always begins with FC or FD and is usually assigned to an interface in addition to its self-assigned link local address
        ∠ Global address also called a global unicast address,can be routed on the internet.These addresses are like IPv4 public IP addresses.The first 48 bits of the address is the global routing prefix.When an ISP assigns a global address to a customer,it's these 48 bits that are assigned.An organization that leases one Global Routing Prefix from it's ISP can use it to generate many IPv6 global address.

    Viewing IP address settings

    1. TCP/IP has assigned the Ethernet connection two IP addresses:One IPv4 address and One IPv6 address
    2. The wireless LAN connection on a different subnet has also been assigned an IPv4 address and an IPv6

    IPv6 addressing is designed so that a computer can self-configure it's own link local IP address which is similar to how IPv4 uses APIPA,

      1. the computer creates it's IPv6 address by using the FE80::64 prefix and uses it's MAC address to generate an interface ID for the last 64 bits
      2. It then performs a duplicate address detection process to make sure it's IP address is unique on the network
      3. It then asks if DHCPv6 server is available on the network to provide configuration information.If DHCPv6 responds with IP address or default gateway's IP address,the computer uses it.It also keeps it's original link local address.DHCPv6 server serves up only global or unique local address

    Tcp and Udp delivery methods

    TCP or UDP manages communication with the application protocols as well as the protocols in the lower layers,which control communication on the network.

    TCP guaranteess delivery
      TCP Guarantees message delivery
      TCPmakes a connection,sends the data,checks whether the data is received and resends if it is not send
      TCP is also called connection-oriented protocol
      TCPis used by applications such as emai and web browsers and it is used when it is important that the data reaches it's destination
      It uses protocols at the internet layer to establish a session betweenclient and server to verify that communication has taken place.When TCP message reaches it's destination,an acknowledgement is sent back to the source.If the source doesn't receive the acknowledgement,it resends the data or passes an error message back to the higher-level application protocol
    UDP provides fast transmissions
      User Datagram Protocol does not guarantee delivery by first establishing connection or by checking whether data is received
      UDP is used for broadcasting such as streaming live video or sound over the web and is also used to monitor network traffic
    TCP/IP protocols used by applications

    Some applications that that use the internet are web browsers,email.chat,FTP,remote sdesktop and remote assistance.:
    Here is some information used by applications

  • HTTP (Hypertext Transder Protocol) is the protocol used for World Wide Web and web browsers and web servers tocommunicate
  • HTTPS (Http secure) Refers to HTTP protocol working with a security protocol such as Secure Socket Layer (SSL) to create a secured socket.TLS is better than SSL. A socket is a connection between a browser and web server.HTTPS is used by web browsers and servers to secure the socket by encrypting the data before it is sent and then decrypting it on the receiving and before the data is processed
  • SMTP (Simple Mail Transfer Protocol) Is used to send an email message to it's destination.The email server that takes care of sending email message is often referred to as SMTP server
  • POP and IMAP (Post Office Protocon version 3 and Internet Message Access Protocol version 4) After an email arrives at it's destination email server,it remains there until the recipient requested delivery.The recipient email server uses one of the two protocols to deliver the message.Using POP3,email is downloaded to the computer and the email is then deleted from the email server.Using IMAP4, the client application manages the email while it is still stored on the server
  • RDP (Remote Desktop Protocol) Is used by the windows remote desktop and remote assistance utilities to connect to and control a remote computer
  • Telnet is used by telnet client/server applications to allow administrator or other user to control a computer remotely.It is not secure because transmissions in telnet are not encrpted
  • SSH (Secured Shell ) Protocol encrypts communications so hackers can't read the data if they intercept a transmission.SSH is used in various situations for encryption such as remotely controlling a computer or when communicating to a server.SSH is commonly used in Linux to pass signing information to a remote computer and control that computer over a network,it is secure and preferred over Telnet
  • FTP (File Transfer Protocol) Is used to transfer files between two computers over WAN or LAN connection.Web browsers can use the protocol as can File Explorer in woindow,also third party FTP client software such as CuteFTP cutetp.com or open source FileZilla ( filezilla-project.org offer edditional features.FTP transmission is not secure.Two protocols that encrypt FTP transmisions are FTPS (FTP Secure ),which uses SSL encryption and SFTP (SSH FTP),which uses SSH encryption
  • SMB (SErver Massage Block) Is a file access protocol originally developed bby IBM and used by windows to share files and printers on a network.The current release of the SMB protocol is SMB 3.the older version include SMB 2 and a spinoff protocol called CIFS (COmmon Internet File System
  • AFP (Apple Filing Protocol) is a file access protocol used by earlier edition s of the Mac operating system by Apple and is one protocol in the old suite of Apple networkikng protocols called AppleTalk.TCP/IP has replpcaed AppleTalk for most networking protocols in the macOS.Current macOS releases use SMB 3 for file access and supoort both AFP and CIFS for backward compatibility.
  • LDAP (LIghtweight Directory Access Protocol) Is used by various clients applications when thee application needs to query a database.Data is sent and received using LDAP protocol is not encrypted.Encryption layer using SSL is sometimes added to LDAP transmissions
  • SNMP (Simple Network Management Protocol) Is a versitile protocol used to monitor network traffic and manage devices.It can help create logs for monitoring devices and network performance.It can make automatic changes to devices being monitored,and it can be used to alert network technicians when bottleneck or other performance issues are causing problems in the networkThe SNMP server is called the manager,and a small application called an agent is installed on devices being managed by SNMP
